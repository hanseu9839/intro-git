# 함수의 프로토타입 
모든 함수가 내부에서 어떻게 작동하는지 외울수도 없고 알 필요도 없다. 아주 복잡한 함수들을 조합하면서 사용한다. 우리가 알아야할 것은 뭐가 들어가고 뭐가 나오는가만 알면된다. 그런 감각을 익히기위해서 블랙박스로써의 함수를 알아야한다. 입력이 뭔지 출력이 뭔지 알지만 내부는 블랙 즉,뭔지 모르는 것이다. **블랙박스**로써의 **함수를 먼저 디자인을 하고 내부를 채우는 방식**으로 코딩을 해나가는 것이 우리가 코딩하는 방식이다. 
우리가 opensource가 많이 발달해서 블랙박스로써의 함수를 사용하는게 중요하다. 
'''
print_multiple_char('*',WIDTH, true);
'''
print_multiple_chars에 argument로 무엇이 들어가는지에 대해서 알 수 있고 기능이 무엇인지에 대해도 대강 추측을 할수있다. 

컴퓨터가 컴파일을 할 때는 블랙박스로써의 무엇인지에 대해만 알면된다. 입력이 뭐고 출력이 뭔지에 대해서만 알면 컴파일 할 수 있다. 즉, 함수의 프로토타입만 있으면 빌드 할 수 있다.  

'''void print_multiple_char(char c,int n_stars, bool print_newline);
/*
void print_multiple_char(char c,int n_stars, bool print_newline)
{
	for (int i = 0; i < n_stars; i++)
		printf("%c", c);

	if (print_newline)
		printf("\n");
	
}
*/
'''
한줄이 가지고 있는 정보는 함수의 이름과 입력으로 무엇이 들어가는지 반환 값으로 무엇을 하는지 어떤 자료형의 자료형인지를 추측할 수 있다. 이렇게 작성하면 컴파일은 되고 링킹은 안된다. 즉, 컴파일이 프로토타입만 가지고 실행한다.실행파일을 만드는 링킹을 하는 단계에서는  몸체가 어디있구나라는 것을 인식하여 끼워 맞춘다. 링킹에러가 이러한 에러가 나오는 것이다. 몸체가 없으면 실행을 할 수 없다. 프로토타입만 있어도 빌드 즉, 컴파일은 된다는 것을 알 수 있다.

C,C++의 우리의 직관과는 다른 특징이다.

Visual Studio에서 Quick action and Refactoring을 사용하여 그안에 있는 signature은 함수의 특징과 입력이 무엇인지 나타내는 말이다. 
Create Declartion(프로토타입을 선언한다)/ defintion은 (정의) 즉 선언과 정의를 따로한다라는 것이다. 이것을 하면 선언되어진 함수를 선언과 정의를 따로해줄수있다. 

정의되어진 파일들을 파일에서 분리할 수있다. 즉, 파일을 여러개 사용할 수 있다. 아예 눈에 안보이는 곳에서 사용해서 생각을 단순하고 정리가 잘되게 할 수 있다. 

argument라는 개념과 parameter라는 개념이 있다.
